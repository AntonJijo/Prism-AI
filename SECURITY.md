# Security Policy

Security is a core priority for PRISM AI.

---

# Reporting Vulnerabilities

Please do NOT publicly disclose vulnerabilities immediately.

Instead:
- create a private report
- include reproduction steps
- include logs/screenshots when possible

---

# Security Focus Areas

The following areas are security-sensitive:
- orchestration systems
- authentication
- runtime isolation
- local execution
- infrastructure
- APIs
- Android virtualization systems
- cloud execution environments

---

# Responsible Disclosure

We appreciate responsible disclosure and coordinated fixes.

---

# Secrets & Credentials

Never commit:
- API keys
- signing keys
- service role keys
- production secrets
- certificates
- private credentials
